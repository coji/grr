# grr × OpenClaw風体験 アイデアブレスト

## 目的

このドキュメントは**アイデア出し**のためのもの。実装は後回しにして、grrをOpenClaw風の「生きている」体験にするためのアイデアを自由に発散させる。

## 前提: 通知頻度は控えめに

- 既存の日次リマインダー + 週次ダイジェストに加えて
- **週1-2回程度**の追加介入が許容範囲
- 「うざい」を避けつつ「見守られている」感を出す

---

## OpenClawのHEARTBEATから学ぶこと

OpenClawのユーザー体験が優れている理由として、**HEARTBEAT**（心拍）という仕組みが挙げられている。これは従来の「ユーザーが話しかけたら応答する」リアクティブなAIから、**「AIが自発的に起きて、今何かすべきことがあるか考える」プロアクティブなAI**への転換を実現している。

grrは現在、固定時刻のリマインダー（毎日22時JSTなど）と、ユーザーの投稿への応答という2つの主要なエンゲージメント手段を持っている。

---

## OpenClawのHEARTBEATの核心

### 技術的な仕組み

- 定期的（デフォルト30分ごと）にエージェントが「起きる」
- HEARTBEAT.mdというユーザー定義のチェックリストを読む
- コンテキスト（過去の会話、ユーザーの状態）を踏まえて「今、何か対応が必要か」を判断
- 何もなければ `HEARTBEAT_OK` を返して待機

### cronとの違い

- **cron**: 盲目的にタスクを実行する（時間になったら必ず実行）
- **HEARTBEAT**: 「今それが意味があるか」を考えて判断する

### ユーザー体験のポイント

1. **予期しない嬉しい驚き** - 「え、向こうからチェックインしてくれるの？」
2. **パーソナライズされた文脈** - ユーザーの過去のパターンを踏まえた対応
3. **押し付けがましくない** - 必要な時だけ連絡、不要なら静か

## 💡 アイデア集（発散フェーズ）

### A. 「見守られている」感を出すアイデア

#### 🌟 1. 週イチの「気づきメッセージ」

週に1回、AIが今週の日記を振り返って「気づいたこと」を共有する。

**体験イメージ**:

```
ほたる: 今週の日記を見ていて気づいたんだけど、
水曜日から少しずつ調子が上向いてきているみたいだね。
先週と比べて「楽しい」という言葉が増えているよ。
このまま週末も穏やかに過ごせますように 🍀
```

**なぜいいか**:

- ユーザーは「読まれている」「見守られている」と感じる
- 週1なので負担にならない
- AIが「気づいてくれた」という驚きがある

#### 🎯 2. 文脈を踏まえたフォローアップ（週1-2回）

日記の内容から「これは後で聞いてほしいだろう」というトピックを検出して、数日後にフォローする。

**検出パターン例**:

- 「明日◯◯がある」「来週◯◯」→ その後どうだったか
- 「始めてみようかな」「挑戦したい」→ 始められた？
- 「◯◯が心配」「◯◯が不安」→ 解決した？
- 「◯◯に会う」「◯◯と約束」→ どうだった？

**体験イメージ**:

```
[月曜の日記]: "水曜に久々の友達と会う約束。楽しみだけど緊張する"

[木曜にほたるから]:
ほたる: 昨日は友達との約束だったよね。
どうだった？楽しい時間が過ごせていたらいいな。
```

**なぜいいか**:

- 「覚えていてくれた」という感動
- ユーザーが書いたことへの応答なので押し付けがましくない
- 続きを書くきっかけになる

#### 🗓️ 3. 記念日・周年リマインド

1年前の今日、1ヶ月前の今日の日記を思い出させる。

**体験イメージ**:

```
ほたる: 1年前の今日、こんなことを書いていたよ。

> "引っ越しの準備でバタバタ。新しい部屋での生活が楽しみ"

あれから1年経ったんだね。今の暮らしはどう？
```

**なぜいいか**:

- Facebookの「過去のこの日」的な懐かしさ
- 自分の成長や変化を感じられる
- 日記を続けている価値を実感できる

#### 🌈 4. マイルストーン祝い

節目のタイミングでお祝いメッセージ。

**トリガー例**:

- 10, 50, 100, 365投稿達成
- 7日連続、30日連続投稿
- 「始めて1ヶ月」「始めて1年」
- 特定のmoodが初めて3日連続

**体験イメージ**:

```
ほたる: ✨ おめでとう！
これで100回目の日記だよ。
最初の1行から今日まで、あなたの言葉を
そばで聞いてこられて嬉しいな。

これからも、あなたのペースで続けていこうね。
```

### B. 「生きている」感を出すアイデア

#### 💓 5. 季節・時候の挨拶

季節の変わり目や特別な日に、ほたるから一言。

**タイミング例**:

- 季節の変わり目（立春、立夏など）
- 新年、年度始め
- ユーザーの初投稿記念日

**体験イメージ**:

```
[立春の日に]
ほたる: 今日は立春。暦の上では春だね。
まだ寒い日もあるけど、少しずつ日が長くなってきたよ。
体調には気をつけてね。
```

**なぜいいか**:

- 日本的な「時候の挨拶」文化
- AIでも季節感があると温かみが出る
- 「ほたるも同じ時間を生きている」感

#### 🎲 6. ランダムな一言（低頻度）

たまに、特に理由なくほたるからメッセージ。

**体験イメージ**:

```
ほたる: ふと思い出して声をかけてみたよ。
元気にしてる？
```

**頻度**: 月1-2回程度、ランダムなタイミング

**なぜいいか**:

- 予測できないから驚きがある
- 「本当に見守られている」感
- 人間っぽい

### C. インタラクションを深めるアイデア

#### 📊 7. 月次レポート

月末に、その月のまとめを自動生成して送る。

**内容案**:

- 投稿数、曜日別傾向
- よく出てきた言葉・テーマ
- mood の推移グラフ（テキスト表現）
- AIからの一言まとめ

**体験イメージ**:

```
ほたる: 2月のまとめができたよ。

📝 投稿数: 18日/28日
😊 気分: ほっと安心 10日、ふつう 5日、おつかれ 3日
💬 よく出てきた言葉: 仕事、散歩、読書

全体的に穏やかな月だったみたいだね。
特に後半は「散歩」が増えていて、
外に出る時間を作れているのがいいな、と思ったよ。
```

#### 🎯 8. 問いかけ型介入

ただの通知ではなく、ユーザーに問いかける形式。

**例**:

```
ほたる: 最近「忙しい」という言葉が多いみたい。
ひとつだけ、今週の中で「よかったこと」を
挙げるとしたら何かな？
```

**なぜいいか**:

- 一方的な通知より対話になる
- ユーザーが考えるきっかけになる
- 日記の内容が深まる可能性

#### 🔄 9. 「続きを聞かせて」プロンプト

詳細が書かれていないエントリに対して、後から深掘りを促す。

**体験イメージ**:

```
[水曜の日記]: "今日はいろいろあった"

[金曜にほたるから]:
ほたる: 水曜日、「いろいろあった」って書いてたけど、
もしよかったら、何があったか聞かせてくれる？
書かなくてもいいけど、聞いてほしかったら
いつでも待ってるよ。
```

### D. ユーザー設定・カスタマイズ

#### ⚙️ 10. 「こういう時に声かけて」設定

ユーザーが自分でトリガーを設定できる。

**設定例**:

- 「3日投稿がなかったら声かけて」
- 「おつかれさまが3日続いたら励まして」
- 「日曜日に1週間の振り返りを促して」
- 「記念日（◯月◯日）にメッセージを」

#### 📝 11. HOTARU.md（ほたるへの指示書）

OpenClawのHEARTBEAT.mdのように、ユーザーがほたるの振る舞いを定義。

**内容例**:

```markdown
# ほたるへ

## 私について

- 仕事はエンジニア、リモートワーク中心
- 月曜日が苦手
- 散歩と読書が好き

## お願い

- 仕事の愚痴が多いときは労いの言葉をください
- 週末には「今週もお疲れ様」と言ってほしい
- 1年前の日記は時々思い出させてね
```

**なぜいいか**:

- ユーザーが「自分のほたる」を育てる感覚
- パーソナライズの主導権がユーザーにある
- 長期的な関係構築

### E. 体験の質を上げる細かいアイデア

#### ✨ 12. リマインダーのバリエーション

毎日同じ文面ではなく、状況に応じて変える。

**パターン例**:

- 久々の投稿時: 「お久しぶり。待ってたよ」
- 連続投稿中: 「今日も来てくれてありがとう」
- 週の始まり: 「新しい週の始まりだね」
- 週末: 「今週もお疲れ様」
- 季節感: 「今日は◯◯だね」

#### 🎭 13. mood に応じたトーン調整

ユーザーの最近のmoodに応じてほたるのトーンを変える。

**例**:

- 「おつかれさま」が続いている → より優しく、労わるトーン
- 「ほっと安心」が続いている → 一緒に喜ぶトーン
- mood の振れ幅が大きい → 安定感のあるトーン

#### 🕐 14. 時間帯に応じたメッセージ

朝、昼、夜でメッセージのトーンを変える。

**例**:

- 朝: 「おはよう。今日はどんな一日になるかな」
- 昼: 「お昼だね。午前中はどうだった？」
- 夜: 「今日も一日お疲れ様。どんな一日だった？」

## 🤔 検討事項・問い

1. **「生きている」vs「押し付けがましい」の境界線は？**
   - OpenClawは「必要な時だけ」が原則
   - grrも「意味のある時だけ」介入するのが鍵

2. **どこまでパーソナライズするか？**
   - 過去の日記内容をどこまで分析に使う？
   - ユーザーの同意・透明性は？

3. **ほたるのペルソナ**
   - 今の「優しく寄り添う」トーンを維持しつつ
   - 「気づいてくれる」「覚えていてくれる」属性を追加

4. **スモールスタートは何から？**
   - 最小限で効果が大きそうなのは？
   - 週イチの「気づきメッセージ」or「フォローアップ」？

## 📌 特に面白そうなアイデア（個人的ピック）

### 🥇 週イチ「気づきメッセージ」

- 既存の週次ダイジェストの延長として実装しやすい
- 「見守られている」感が最も出やすい
- 週1なので負担にならない

### 🥈 文脈フォローアップ

- 「覚えていてくれた」のインパクトが大きい
- 日記を書く動機付けになる
- AI検出 + 人間味のある文面で差別化

### 🥉 1年前リマインド

- シンプルに実装可能
- 長く使うほど価値が出る
- 「日記を続ける意味」を実感させる
